# Maven 本地配置管理客户端使用说明

## 1. 安装与启动

macOS 直接打开打包产物：

```bash
open "src-tauri/target/release/bundle/macos/Maven Settings Management.app"
```

也可以安装 DMG：

```bash
open "src-tauri/target/release/bundle/dmg/Maven Settings Management_0.1.0_aarch64.dmg"
```

开发模式启动：

```bash
npm install
npm run tauri:dev
```

## 2. Maven 环境管理

进入“环境”页后，客户端会自动检测当前 Maven：

- 优先使用已手动保存的 Maven 路径
- 再检测 `MAVEN_HOME`
- 再检测 `M2_HOME`
- 最后从系统 `PATH` 查找 `mvn`

如果自动检测失败，可以手动选择：

- “选择 Home”：选择 Maven 安装目录，例如 `/Users/name/work/apache-maven-3.8.1`
- “选择 mvn”：选择 Maven 可执行文件，例如 `bin/mvn` 或 Windows 下的 `mvn.cmd`
- “使用此路径”：校验路径并保存为当前 Maven

校验成功后会显示：

- Maven 版本
- Maven Home
- `mvn` 路径
- Java 版本
- `mvn -v` 原始输出

## 3. Settings 配置文件管理

进入“配置文件”页，可以管理多套 `settings.xml`。

### 新建配置

- “空模板”：创建一个最小 settings 文件
- “从默认 settings”：读取当前用户默认 `~/.m2/settings.xml` 创建配置

### 导入配置

点击“选择文件”选择任意外部 `settings.xml`，填写显示名称后点击“导入”。

导入的外部文件默认直接管理原路径；删除配置时只会从客户端列表移除，不会删除原文件。

### 配置列表操作

每个配置支持：

- 打开：进入编辑
- 命令：复制 `mvn -s "<settingsPath>"` 命令
- 重命名：修改客户端显示名称
- 复制：复制为一份应用内配置
- 默认：备份并覆盖 `~/.m2/settings.xml`
- 试运行：用当前 Maven 执行离线 `help:effective-settings` 校验
- 删除：删除应用内配置文件；外部导入配置只移出管理列表

## 4. 可视化编辑

可编辑内容包括：

- 基础设置：`localRepository`、`interactiveMode`、`offline`
- 插件组：`pluginGroups`
- 镜像源：`mirrors`
- 代理：`proxies`
- 认证服务：`servers`
- Profiles：`profiles`、属性、仓库、激活状态

点击顶部“保存”后写回当前配置文件。

## 5. XML 源码编辑

进入“XML”页可以直接编辑完整 XML。

规则：

- XML 能解析时会自动同步回表单
- XML 格式错误时会提示错误，并禁止保存
- 表单编辑会保留未可视化的顶层 XML 和 Profile 内高级 XML 片段

这意味着 `activation`、`pluginRepositories` 等暂未做表单化的高级配置，不会因为你改了镜像或 server 被清掉。

## 6. 默认配置与备份恢复

点击“设为默认”时：

1. 如果默认 `~/.m2/settings.xml` 已存在，先自动备份
2. 将当前配置写入默认路径
3. 标记该配置为默认配置

备份会显示在“配置文件”页底部“默认配置备份”区域。

点击“恢复”可以把某个备份恢复到默认 `settings.xml`。恢复前也会先备份当前默认文件。

## 7. 试运行校验

“试运行”会执行：

```bash
mvn -s "<settingsPath>" --offline help:effective-settings -DskipTests
```

用途：

- 验证当前 Maven 是否能读取该 settings
- 验证 XML 与基础 Maven 配置是否有效
- 不主动联网，避免校验时下载依赖

如果本机没有缓存 `maven-help-plugin`，离线试运行可能失败；这不一定代表 settings 文件无效，需要结合输出判断。

## 8. 本地数据位置

应用数据目录：

- macOS: `~/Library/Application Support/Maven Settings Management`
- Windows: `%APPDATA%\Maven Settings Management`
- Linux: `${XDG_DATA_HOME:-~/.local/share}/maven-settings-management`

默认 Maven settings 路径：

- macOS/Linux: `~/.m2/settings.xml`
- Windows: `%USERPROFILE%\.m2\settings.xml`

## 9. 打包命令

macOS：

```bash
npm run package:mac
```

产物位置：

- `src-tauri/target/release/bundle/macos/Maven Settings Management.app`
- `src-tauri/target/release/bundle/dmg/Maven Settings Management_0.1.0_aarch64.dmg`

Windows 官方 NSIS 安装包：

```bash
npm run package:win
```

该命令会使用 `cargo-xwin` 交叉编译 Windows x64 程序，并交给 Tauri 官方 NSIS bundler 生成安装包。

如果卡在下载 `nsis_tauri_utils.dll`，可以改用仓库内置备用脚本：

```bash
npm run package:win:exe
npm run package:win:manual
```

备用脚本位置：

- `packaging/windows/maven-settings-management.nsi`

备用安装包能力：

- 安装到当前 Windows 用户目录：`%LOCALAPPDATA%\Programs\Maven Settings Management`
- 创建开始菜单快捷方式
- 创建桌面快捷方式
- 写入 Windows 卸载项
- 支持卸载

Windows 产物位置：

- exe: `src-tauri/target/x86_64-pc-windows-msvc/release/mvn-settings-management.exe`
- 备用 NSIS 安装包: `src-tauri/target/x86_64-pc-windows-msvc/release/bundle/nsis/Maven Settings Management_0.1.0_x64-setup.exe`

Windows MSI 建议在 Windows 机器或 Windows CI 上打包。
